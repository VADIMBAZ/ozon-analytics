## Цель

Добавить в дашборд раздел юнит-экономики на основе ежемесячных файлов Озон `Юнит-экономика_*.xlsx`. Главные вопросы, на которые отвечает раздел:

1. Как менялась прибыльность каждого SKU по месяцам?
2. Где утекают деньги? (структура расходов)
3. Сколько реальной прибыли мы потеряли из-за стокаутов?

## Принятые решения

| Вопрос | Решение |
|---|---|
| Себестоимость (COGS) | Одна цифра на все SKU (артикулы *SK2024 — варианты одного ключа). Дефолт 25 ₽ (из текущего `COST_PER_UNIT`). |
| Хранение COGS | `st.session_state['cost_per_unit']` — заменяет хардкод-константу, влияет на оба раздела (запасы + юнит-экономика) |
| Размещение поля COGS | Сайдбар, отдельная мини-секция после фильтра периода |
| Структура раздела | Один раздел с табами внутри |
| Размещение раздела | В самом низу, после «Анализа запасов» |
| Реакция на глобальные фильтры | Реагирует на оба: Период + Артикулы |
| Загрузка unit-файлов | Расширить существующий UI-загрузчик, multi-file |
| Папка | `data/unit_economics/` |

## Источник данных

**Имя файла:** `Юнит-экономика_DD.MM.YYYY-DD.MM.YYYY.xlsx`. Период берётся из имени. Только календарные месяцы — произвольные диапазоны skip + warning.

**Структура листа** (фиксированная):
- 4 строки шапки, данные с R5
- 29 колонок: SKU, Артикул, Название, Схема, **Себестоимость** (всегда 0 у нас), Цена, Заказано, Доставлено, Возвращено, 5 групп расходов, итоговые поля (Прибыль за шт, Прибыль за период, Доля от продаж, Рекомендация)

**Группы расходов:**

| Группа | Поля |
|---|---|
| Продажи | Выручка, Баллы за скидки, Программы партнёров, **Вознаграждение Ozon**, Эквайринг |
| Доставка | Обработка отправления, Логистика, Доставка до ПВЗ, Размещение |
| Возвраты | Обработка возврата, Обратная логистика |
| Доп. услуги | Утилизация, Ошибки продавца |
| Реклама | Оплата за клик, Оплата за заказ |
| Итоги | Доля от продаж, Прибыль за шт, Прибыль за период, Рекомендация Ozon |

**Парсер**: `load_unit_economics() → pd.DataFrame` с `period_start`, `period_end` и всеми финансовыми полями. Один артикул × один период = одна строка. `@st.cache_data` по mtime списка файлов.

## Ключевые формулы

```
Прибыль за шт (Озон) = Цена × Доля_от_продаж
Прибыль за период (Озон) = Прибыль_за_шт × (Доставлено − Возвращено)

COGS итого = cost_per_unit × (Доставлено − Возвращено)
Чистая прибыль = Прибыль_за_период − COGS_итого
Маржа чистая % = Чистая_прибыль / Выручка × 100
Упущенная прибыль = Упущенная_выручка × Маржа_чистая %
```

## UI компоновка

```
─── 💰 Юнит-экономика ───
ℹ️ Финансовая раскладка по данным Озон.
   Себестоимость задаётся в сайдбаре (сейчас 25 ₽).

┌── KPI за период (с дельтами vs предыдущий период) ──┐
│  Выручка | Прибыль до COGS | Прибыль чистая | Маржа % │
└────────────────────────────────────────────────────────┘

┌── Структура расходов (% выручки) ──┐
│  Комиссия | Логистика | Реклама | Возвраты │
└────────────────────────────────────────────┘

╔═══ 📈 Динамика маржи ═══ 💸 Структура расходов ═══ 🎯 Упущенная прибыль ═╗
║                                                                          ║
║  (содержимое таба)                                                       ║
║                                                                          ║
╚══════════════════════════════════════════════════════════════════════════╝
```

### Таб 1: 📈 Динамика маржи (дефолтный)

- **Линии**: Маржа % помесячно — агрегат + по SKU (цвета из общей палитры)
- **Бары**: Прибыль ₽ помесячно (стек по SKU)
- **Heatmap**: SKU × месяц, цвет = маржа %

### Таб 2: 💸 Структура расходов

- **Waterfall** по агрегату или по выбранному SKU: Выручка → −Баллы → −Комиссия Озон → −Эквайринг → −Логистика → −Возвраты → −Реклама → −COGS → ▮ Чистая прибыль
- **Стек-бар 100%**: структура расходов в % выручки помесячно

### Таб 3: 🎯 Упущенная прибыль

- Таблица: `SKU | Упущенная выручка | Маржа % | Упущенная прибыль`
- Итог: «Из-за стокаутов потеряли ~N ₽ реальной прибыли»

## Edge cases

| Ситуация | Поведение |
|---|---|
| Нет unit-файла за месяц из analytics | Пропуск месяца, warning |
| Произвольный диапазон в файле | Skip + warning |
| Пустой период | Warning, графики не рисуются |
| COGS = 0 | Чистая прибыль = маржинальный вклад, без предупреждений |
| COGS > Цена | Warning, отрицательная маржа отображается как есть |
| Нет предыдущего окна | Дельты KPI скрыты |
